# Open Source Respect - Public Short Version

Use this short version in public README files.

## Our stance

We value open source maintainers and contributors as people, not as a free service desk.

## What we expect

- Respectful communication in issues, PRs, and discussions.
- Reproducible bug reports (steps, versions, logs).
- Constructive review behavior focused on code and facts.
- Realistic expectations on response times and roadmap ownership.

## What we reject

- Harassment, insults, and personal attacks.
- Public pressure or shaming of volunteer maintainers.
- Demanding unpaid SLA-like support.

## For business users

If a feature/fix is business-critical, consider sponsoring or implementing it with a contribution.
