# Code of Conduct

## Purpose

This project is a professional engineering workspace. We want collaboration to be respectful, factual, and safe for everyone.

## Expected Behavior

- Communicate respectfully, even under pressure.
- Focus feedback on code, behavior, and facts, not on people.
- Document decisions and tradeoffs clearly.
- Escalate risks early (security, production impact, data loss, compliance).
- Accept and give review feedback constructively.

## Unacceptable Behavior

- Harassment, discrimination, insults, or personal attacks.
- Threatening language or intimidation.
- Deliberate sabotage, hiding critical risks, or knowingly bypassing controls without approval.
- Sharing sensitive data (credentials, secrets, personal data) in code, logs, or chat.

## Security and Production Safety

- Production-impacting changes must include risk and rollback notes.
- Never commit secrets or tokens.
- If a change may cause service disruption, pause and request peer review.

## Reporting

If you experience or observe unacceptable behavior, contact:

- Team lead / service owner
- Project maintainers (via PR comment or internal channel)
- Security/compliance contact if the issue is security-related

Reports should include what happened, where, and when. All reports are handled confidentially as far as possible.

## Enforcement

Maintainers may edit/remove comments, reject contributions, or restrict participation for behavior that violates this policy.

## Scope

This code of conduct applies to:

- Repository discussions, issues, PRs, and commit messages
- CI/CD discussions and release coordination
- Internal collaboration channels related to this project
